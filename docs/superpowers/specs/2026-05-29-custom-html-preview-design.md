# Custom HTML 预览工具设计

## 背景

`/custom-html` 当前只有一个黑底双栏骨架。最新需求明确页面需要同时展示：

1. 左侧 PC 端预览。
2. 右侧手机端预览。
3. 一个用于放置源码的配置项。

经过可视化方案比较后，确认采用三栏驾驶舱布局：左侧源码配置、中间 PC 预览、右侧手机预览。

## 目标

为 `/custom-html` 提供一个可直接使用的 HTML 双端预览工作台：

1. 左侧常驻源码配置面板，支持编辑 HTML/CSS 源码。
2. 中间展示 PC 端主预览，作为主要检查区域。
3. 右侧展示手机端预览，用固定手机机身承载同一份源码。
4. 顶部提供核心操作与状态信息，让用户能快速加载示例、刷新、复制、清空。

## 非目标

以下能力不在本次范围内：

1. 不接入外部文件系统或远程保存。
2. 不提供 Monaco 编辑器、语法高亮编辑能力或格式化器。
3. 不做多设备尺寸管理，只提供一个 PC 自适应画布和一个手机宽度预览。
4. 不解析 HTML 结构并生成可视化配置表单。
5. 不提供历史记录、撤销栈、导出图片等增强能力。

## 页面结构

桌面端使用三栏布局：

1. 源码配置面板
   - 固定宽度，位于左侧。
   - 包含标题、行数/字符数、源码输入框、复制源码、清空源码按钮。
   - 使用深色代码面板风格，强化“编辑区”属性。

2. PC 端预览
   - 位于中间，弹性占据主要宽度。
   - 预览外层使用浅色网格背景，模拟设计画布。
   - iframe 以 `100%` 宽度展示同一份源码。

3. 手机端预览
   - 位于右侧，固定窄栏。
   - 使用手机机身边框包裹 iframe。
   - iframe 宽度按手机视口设计，便于快速检查移动端布局。

窄屏下三栏纵向堆叠，顺序为：源码配置、PC 预览、手机预览，避免横向滚动。

## 交互

1. 默认加载一段示例源码，用户进入页面即可看到 PC 与手机预览差异。
2. 修改源码后，预览文档由同一份源码生成。
3. `刷新预览` 用于显式同步 iframe 内容。
4. `加载示例` 会恢复默认源码。
5. `复制源码` 将当前源码写入剪贴板。
6. `清空` 会清空源码，两个预览区进入空状态。
7. 左侧面板支持折叠，用于在桌面端给 PC 预览让出更多空间。

## 状态与安全

1. 源码为空时，iframe 使用空状态文档，提示用户在左侧输入源码。
2. iframe 使用 `sandbox` 属性隔离预览内容，避免源码影响外层应用。
3. 源码通过 `srcDoc` 注入，不使用 `dangerouslySetInnerHTML`。
4. 页面不执行网络请求，所有状态都保存在当前 React 组件内。

## 测试

新增纯函数测试，覆盖：

1. 源码行数与字符数统计。
2. 非空源码生成预览文档。
3. 空源码生成空状态文档。

页面实现完成后运行：

1. `pnpm test`
2. `pnpm build`
3. 浏览器访问 `http://localhost:3124/custom-html` 做桌面视口检查。
